# UX Wireframe Specifications

## Global navigation

- Home Dashboard
- Case Portal
- Counselor Workspace
- School Overview
- District Overview
- Analytics
- Reports
- Notifications
- Admin Settings

## Screen: Case Portal

Primary layout:

- Left filter rail: school, priority, category, status, owner, age bucket
- Center case table: case ID, priority, category, school, submitted date, SLA status, owner, current status
- Right quick action panel: assign, escalate, create follow-up, export

## Screen: Case Details

Sections:

- Header: case ID, status, priority, SLA indicator, assigned owner
- Summary: category, school, intake channel, submitted date
- Timeline: status changes, notes, follow-ups, escalations
- Actions: add note, update status, create follow-up, escalate, close
- Audit panel: field changes and user actions

## Screen: Student Timeline

- Uses anonymized student key
- Displays prior cases by term and category
- Does not display diagnosis, clinical risk score, or medical prediction
- Supports operational repeat incident analysis only

## Screen: Counselor Workspace

- My active cases
- Due today
- Overdue follow-ups
- SLA at risk
- Calendar-style follow-up view
- Workload summary

## Screen: District Overview

- Executive KPI strip
- Region/school heatmap
- Monthly trend
- Category mix
- School performance index ranking
- Staffing recommendation panel

## Screen: Admin Portal

- User and role management
- SLA policy configuration
- Category management
- School and department reference data
- Export permissions
- Audit log search
