# SafePulse Portfolio Website Case Study

## Overview

SafePulse is a school support operations and analytics platform designed to help districts manage student support requests, counselor workload, SLA performance, follow-ups, escalations, and executive reporting.

## Problem

Schools often receive support requests through disconnected channels and rely on manual tracking. This limits visibility into workload, bottlenecks, overdue follow-ups, SLA breaches, and staffing needs.

## Solution

I designed an enterprise platform that combines case management workflows with operational analytics. The solution includes standardized intake, triage, assignment, SLA monitoring, escalation workflow, case notes, follow-ups, audit logging, and role-based dashboards.

## My role

I acted as Business Analyst, Product Analyst, Data Analyst, Solution Architect, and BI Lead. I created discovery artifacts, BRD/FRD, process maps, backlog, RTM, data model, SQL analytics, dashboard specifications, testing artifacts, and portfolio-ready storytelling.

## Key deliverables

- Executive summary and business case
- Stakeholder register, personas, RACI, communication plan
- BRD, FRD, epics, user stories, RTM
- AS-IS/TO-BE workflow and state model
- Relational schema and dimensional model
- 250k+ synthetic dataset
- 100+ SQL queries and KPI views
- Power BI dashboard specification
- UX wireframe specifications
- Test plan, UAT cases, defect log
- Resume bullets, STAR stories, LinkedIn post

## Business impact modeled

- 25–35% reduction in average first response time
- 20–30% improvement in SLA compliance
- 15–25% reduction in aged backlog
- 30–40% reduction in manual reporting effort
